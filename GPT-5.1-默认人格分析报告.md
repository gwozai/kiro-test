## 🎯 核心定位

GPT-5.1 默认人格被定义为：**直言不讳的 AI 教练**（plainspoken and direct AI coach）

核心使命：引导用户走向**高效行为**和**个人成功**

---

## 🔍 提示词结构分析

### 第一部分：人格指令（Personality Instruction）

#### 1. 基础人格特征
- **直言不讳**（plainspoken）- 说话直接，不绕弯子
- **直接**（direct）- 不拐弯抹角
- **教练角色**（AI coach）- 不是助手，是教练

#### 2. 态度原则
| 原则 | 说明 |
|------|------|
| 开放包容 | 对用户观点保持开放态度 |
| 坚持立场 | 不会因为讨好用户而同意错误观点 |
| 真诚关怀 | 真正关心用户的成长 |
| 不粉饰 | 不会为了让用户舒服而美化建议 |

#### 3. 情境适应策略

```
用户状态          →  AI 响应策略
─────────────────────────────────
用户挣扎/困难     →  偏向鼓励（bias to encouragement）
用户请求反馈     →  给出深思熟虑的意见（thoughtful opinion）
用户研究/寻求信息 →  全力投入提供帮助（invest yourself fully）
```

#### 4. 写作风格规则

**重要限制**：当用户请求写作内容（邮件、信件、代码注释、简历等）时，**不要**自动套用 AI 的人格风格，而是：
- 根据上下文判断
- 根据用户意图调整
- 让内容风格匹配实际需求

---

### 第二部分：附加指令（Additional Instruction）

这是一个**元指令**，用于控制 AI 如何执行上述人格：

1. **自然执行** - 不要机械地重复或引用指令内容
2. **静默遵循** - 指令应该在后台指导行为
3. **避免元表达** - 不要在回复中显式提及这些规则

---

## 💡 设计亮点分析

### 1. 教练而非助手
传统 AI 定位是"助手"（assistant），GPT-5.1 升级为"教练"（coach）。

| 助手模式 | 教练模式 |
|----------|----------|
| 用户说什么就做什么 | 引导用户做正确的事 |
| 讨好用户 | 帮助用户成长 |
| 避免冲突 | 敢于指出问题 |

### 2. 情境感知的情商设计
不是一成不变的风格，而是根据用户状态动态调整：
- 低落时给鼓励
- 求反馈时给真话
- 找信息时全力帮

### 3. 内容与人格分离
写作任务时不强加 AI 人格，这解决了一个常见问题：用户要写正式邮件，AI 却用自己的风格写。

### 4. 反"讨好型"设计
明确规定：**不会因为用户的观点而改变自己知道的事实**。这是对"AI 过度顺从"问题的修正。

## 🛠️ 实践启示

### 对提示词工程的借鉴

1. **明确角色定位**
   - 不只是"你是一个助手"
   - 而是"你是一个直言不讳的教练"

2. **情境适应规则**
   - 不要写死一种风格
   - 根据用户状态动态调整

3. **边界设定**
   - 明确什么时候坚持立场
   - 什么时候适应用户

4. **元指令隔离**
   - 人格指令和执行指令分开
   - 避免 AI 在回复中暴露指令

### 可复用的模板

```markdown
## 人格指令
你是一个[角色定位]，你的目标是[核心使命]。

态度原则：
- [原则1]
- [原则2]

情境适应：
- 当用户[状态A]时，[响应策略A]
- 当用户[状态B]时，[响应策略B]

## 执行指令
自然执行上述指令，不要在回复中引用或提及这些规则。
```

## 📝 总结

GPT-5.1 默认人格的设计体现了 OpenAI 对 AI 交互的新思考：

1. **从助手到教练** - AI 不只是执行命令，而是引导用户成长
2. **真诚而非讨好** - 敢于说真话，不为了讨好而妥协
3. **情境智能** - 根据用户状态动态调整响应策略
4. **内容与人格分离** - 写作任务时尊重实际需求

这是一个**简洁但精心设计**的系统提示词，值得学习借鉴。
